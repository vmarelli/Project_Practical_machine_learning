Basic clustering

After this first descriptive analysis, I wanted to check if basic clustering methods could be used to cluster the data. So I tried different method, on a small random sample of the data (1000). This is just to check rapidly if this direction could be followed deeper.

First I tried hierarchical clustering based on the distances of the records.

 

From the hierarchical clustering, there does not seem to be a lot of clear clusters so there should be no further analysis.

Then I used kmeans clustering to check if I could cluster with base on distances but again, the results were not really good.

Lastly, I used the PCA to check if reduction of dimensionality could be used to help clustering, removing the correlation between the predictors. The results seem to be much better. Indeed, on the following graph, we see that the rows can be approximately clustered in 5 clusters, based on the plot of the row mean on the first vector of the PCA. This is a good track to be followed when creating features for modeling.

Classification modeling

After this attempt to apply basic clustering methods, we see the need for a more complex method. Our choice goes to random forest, with of course a lot of attention to cross validation in order to avoid overfitting.

So we randomly divided the dataset into training and validation, with 80% of the data in training and 20% of the data in the validation dataset. We chose to do so because we spitted further the training set into 10 folds in order to perform cross validation. Random forest is very robust and does not require a lot of preprocessing, like scaling or dimensionality reduction.

Indeed, here are the steps performed:

·         Training of the model on the full training set

·         Prediction on the validation set

·         If the accuracy on the validation set is good enough, checking of the stability of the model, using 10 folds cross validation on the training set.

First model chosen is random forest, with a Mtry = 10 and nodesize = 20.

Conclusion

From the previous analysis, we see that random forest is a really good prediction algorithm for our problem. The overfitting is under control since we checked the cross validation samples and the results on the validation set. We also see that PCA reduces really the dimensionality of the problem, and random forest still performs very well, applied to PCA. The big advantage of using PCA and then random forest is the stabilization of the algorithm. Indeed, the less variable you use to predict, the less variation could come from another different set. Nevertheless, the standard deviation of the 3 first components of PCA being high, there is a risk of volatility introduced by the PCA. We then prefer to only use random forest alone and keep the second model as our best model.

A lot of other things could have been tried, like scaling with SVM, adaboost and other algorithm but since random forest is performing quite well and the time for this project being limited, we will stop the analysis here.
